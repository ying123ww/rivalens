系统逻辑架构

Rivalens 的核心不是让底层 research engine 直接写报告，而是用 Agent DAG 把证据、行业 schema、竞品知识、分析 claim 和报告文档分层生产。

当前 DAG 流转

这是当前代码中的主流程节点。`SchemaBuilderAgent` 和 `ProductFact` 已经从主线移除。

当前核心 Schema

当前主线是 `ActiveKnowledgeSchema -> CompetitorKnowledge -> AnalysisClaim`。报告层的 `ReportDocument` 是下一步。

ActiveKnowledgeSchema

本次任务冻结后的 schema，包括候选行业、选中行业、core fields、industry extensions 和 candidate extensions。

CompetitorKnowledge

每个竞品的结构化知识对象，包含 feature tree、pricing model、user personas、industry extensions 和 evidence ids。

AnalysisClaim

基于 CompetitorKnowledge 生成的分析结论，必须绑定 evidence ids、reasoning 和 confidence。
