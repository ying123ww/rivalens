当前代码主线：focused / verification research control

Rivalens 可追溯竞品分析工作流

维度 ID 已由 planning 阶段确认，collection 直接按 competitor x dimension 执行 focused evidence collection。 覆盖缺口由 CoverageReviewer 生成结构化补采任务；claim 证据不足时再进入 verification。

Planning

确认分析维度

PlanningAgent 生成 10 个 confirmed analysis dimensions，并写入 CompetitorAnalysisState.analysis_dimensions。

Collection

直接采集证据

CollectionAgent 为每个 competitor x dimension 建 root branch， 默认使用 search_stage=focused 并产出 EvidenceItem。

Review

覆盖驱动补采

EvidenceQualityReviewer 过滤证据，CoverageReviewer 根据 expected source types 和 guiding questions 生成 focused follow-up tasks。

Workflow

  1. scope_planner: 规范竞品输入，冻结 schema，确认维度。
  2. source_collection: focused collection，证据 review，coverage review，按预算补采。
  3. knowledge_structuring: 将 accepted evidence 结构化为 competitor knowledge。
  4. dimension_analysis: 基于结构化知识和 accepted evidence 生成 traceable claims。
  5. claim_support_review: 对 weak / unverifiable claims 触发一次 verification pass。
  6. report_writer 和 publisher: 输出报告与证据索引。
Traceability

证据链

  • EvidenceItem 保留 URL、branch、collection task、research task 和 dimension metadata。
  • AnalysisClaim 必须绑定 accepted evidence IDs。
Control

阶段边界

  • focused 是默认 evidence collection。
  • verification 只服务 claim support review 的补证。
Observability

可回放状态

  • research_tasks、coverage_assessments、research_artifacts 保留执行轨迹。
  • agent_events 汇总输入、输出、预算和覆盖情况。
